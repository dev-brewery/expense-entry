Correct Behavior: If an expense is added for November, it should return the total for that category for the month of November. If an expense is added for December, it should return the total for that category ONLY for the month of December.

FIX Required:

Root Cause:
- Date timezone mismatch between form parsing and database storage
- Form dates like "2025-12-01" were parsed as UTC midnight (2025-12-01T00:00:00Z)
- In PST timezone (UTC-8), UTC midnight = November 30th 4:00 PM local time
- This causes dates to shift backward by one day, making December 1st appear as November 30th
- Month extraction then returns November instead of December
